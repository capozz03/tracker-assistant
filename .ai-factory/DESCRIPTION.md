# Tracker Assistant

## Обзор

Минималистичный Python-клиент для Timetta — сервиса управления задачами. Предоставляет CLI-инструмент и библиотечный API для создания задач, управления пользователями, тегами и вложениями через Timetta OData v4 REST API.

## Основные возможности

- **Список проектов** — получение всех проектов из Timetta API
- **Создание задачи** — с названием, описанием, тегами, исполнителем, комментариями и вложениями за один вызов
- **Обновление задачи** — изменение исполнителя и тегов
- **Комментарии** — добавление комментариев к задаче
- **Вложения** — прикрепление файлов к задаче
- **Список пользователей** — получение пользователей с кешированием
- **Список тегов** — получение тегов с кешированием
- **Обогащение задачи** — `enrich-task` форматирует сырое описание через `claude -p`, подбирает теги и исполнителя
- **Verbose-логирование** — все HTTP-запросы в DEBUG-режиме

## Технологический стек

- **Язык программирования:** Python 3.10+
- **Фреймворк:** нет (только стандартная библиотека)
- **CLI:** argparse
- **Менеджер пакетов:** uv + hatchling
- **Внешний API:** Timetta OData v4 (`https://api.timetta.com/odata`)
- **Аутентификация:** Bearer token (статический токен из `.env`)
- **Тестирование:** pytest
- **База данных:** нет

## Архитектурные заметки

Трёхслойная архитектура без зависимостей от внешних библиотек:

1. **CLI-слой** (`scripts/task_cli.py`, `scripts/enrich_task.py`) — парсинг аргументов, форматирование вывода
2. **Pipeline-слой** (`src/tracker_assistant/pipeline.py`) — оркестрация бизнес-логики
3. **Adapter-слой** (`src/tracker_assistant/adapters/timetta_adapter.py`) — HTTP-взаимодействие с Timetta API

## Нефункциональные требования

- **Логирование:** stdlib logging, настраивается через `--log-level`
- **Обработка ошибок:** `SystemExit` для фатальных ошибок конфигурации, `logging.warning` для мягких сбоев
- **Безопасность:** токен только из `.env`, никогда не в исходном коде
- **Изоляция:** сервис работает только в своей директории, нет выхода в родительские пути
- **Кеширование:** пользователи и теги кешируются локально для снижения нагрузки на API
