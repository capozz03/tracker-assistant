## Контекст: где может теряться тег

Пайплайн тегов (submit, основной путь бота):
`build_prompt` (даёт claude список `id=<uuid> name=<имя>`) → claude возвращает
`tags` → `create_tasks` → `resolve_tags(raw, known)` → PATCH в Timetta.

`resolve_tags` ([submit/prompt.py](../../src/tracker_assistant/submit/prompt.py)) матчит: UUID passthrough → имя (lower) → код (lower). Непопавшее — отбрасывается с `WARN [TAG] Тег не найден`.

**Ведущие гипотезы (проверить в Фазе 1):**
1. **Несовпадение имён.** Правила промпта хардкодят «Фронтенд»/«Бекенд», но реальные
   имена тегов в Timetta другие (англ., иное написание, «Бэкенд» через Э, иной кейс) →
   `resolve_tags` не матчит → пустой тег.
2. **Claude выводит англ.** "frontend"/"backend" — `resolve_tags` их НЕ резолвит к русским
   именам (подтверждено `test_english_name_not_resolved`).
3. **Пустой справочник тегов.** `get_tags()` (неверный `TIMETTA_TAGS_DIR_ID`) → в промпте
   «(теги не найдены)» → claude нечего выбрать.

## Примечания по реализации

### Диагностика — обязательна первой
Корень пока не подтверждён. Задача 1 решает, какие из задач 2–4 реально нужны:
- Гипотеза 1/2 (несовпадение имён / англ.) → задачи 2 + 3 (основной фикс).
- Гипотеза 3 (пустой справочник) → задача 4 (и проверка `TIMETTA_TAGS_DIR_ID`).
Не приступать к фиксу, пока `[TAG]`-логи и `list-tags` не покажут фактическую причину.

### Минимальные изменения
Не переписывать весь промпт — точечно поправить правило про слойные теги и
требование выводить UUID. `resolve_tags` расширять аддитивно (новые fallback'и
поверх текущей логики), не ломая UUID/имя/код-матчинг.

### Существующий тест `test_english_name_not_resolved`
Сейчас он фиксирует, что «frontend» НЕ резолвится. После фикса слойные англ.
синонимы ДОЛЖНЫ резолвиться — этот тест нужно переформулировать (не удалять
проверку «мусорный тег → []», а сменить пример на действительно неизвестный тег).

### Граница submit vs enrich
Основной путь бота — `submit_requirements`/`generate_tasks`. `enrich_task` —
отдельный CLI (`task-enrich`); правим его промпт зеркально для консистентности,
но приоритет — submit.
