# AttributeError: forward_date removed in PTB v20+

**Date:** 2026-05-25 19:05
**Files:** src/tracker_assistant/telegram/handlers.py, tests/test_telegram_handlers.py
**Severity:** medium

## Problem

`AttributeError: 'Message' object has no attribute 'forward_date'` при пересылке
сообщения в бот. Исключение возникало в `_extract_forwarded_text` на строке
`if not message.forward_date:`.

## Root Cause

В python-telegram-bot v20 атрибут `forward_date` был признан устаревшим и заменён
на `forward_origin` (объект `MessageOrigin`). В PTB v22+ атрибут окончательно удалён.
Проект использует PTB 22.7, поэтому обращение к `forward_date` вызывало `AttributeError`.

## Solution

Заменил `message.forward_date` на `message.forward_origin` в
`_extract_forwarded_text`. Обновил тесты и хелпер `make_mock_update`:
`forward_date` → `forward_origin`, в тесте `test_forwarded_returns_text` вместо
`datetime(2024, 1, 1)` используется `MagicMock()` (любой truthy объект `MessageOrigin`).

## Prevention

- При обновлении PTB проверять CHANGELOG на removed attributes.
- Не полагаться на deprecated API — `forward_date` был помечен как deprecated с v20.
- Тест на пересылку сообщения должен использовать `forward_origin`, а не `forward_date`.
