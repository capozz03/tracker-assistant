# Плейсхолдер project_id уходит в Timetta и даёт 400 «Entity cannot be null»

**Date:** 2026-05-28 16:23
**Files:** src/tracker_assistant/telegram/handlers.py, tests/test_telegram_handlers.py
**Severity:** medium

## Problem

Все пересланные в бота постановки задач падали с ошибкой
`Timetta POST /Issues → 400: {"message":"The input was not valid... Entity cannot be null","target":"entity"}`.
Пользователь видел криптовую 400 от Timetta вместо понятного сообщения.

## Root Cause

В `.env` стоит `TIMETTA_PROJECT_ID=your_timetta_project_uuid` — плейсхолдер из
шаблона, не настоящий UUID. `telegram_projects.json` отсутствует, поэтому проект
«default» берёт этот плейсхолдер. Когда пользователь не выбрал проект через
`/project`, `_active_project` отдаёт default, и в Timetta уходит
`projectId: "your_timetta_project_uuid"`. Timetta не может связать сущность с
несуществующим проектом и возвращает generic-ошибку «Entity cannot be null»
(та же формулировка, что при отсутствии `typeId` — см. docs/timetta-quirks.md).

Защита в `handle_text` / `_handle_media_message` проверяла только `if not project.project_id`
(пустую строку), а непустой плейсхолдер проскакивал. Это нарушало философию
проекта: «нет валидного project_id → СТОП + понятное сообщение».

## Solution

Добавлен хелпер `_is_valid_project_id()` (через `uuid.UUID(...)`), который
отвергает пустые значения и любые не-UUID (включая плейсхолдер). Проверки в
`handle_text` и `_handle_media_message` заменены с «пусто» на «не валидный UUID»;
при провале — `❌ Проект не настроен. Выберите проект через /project` + WARN-лог
`[FIX]` с фактическим значением. Сабмит в Timetta не вызывается.

## Prevention

- Любое значение из `.env`/конфига, которое уходит во внешний API как ID,
  валидировать на формат у границы (UUID-проверка), а не доверять «непустоте».
- Шаблонные плейсхолдеры (`your_*`) трактовать как «не задано».
- Криптовые ошибки внешних API перехватывать и заменять на actionable-сообщение
  для пользователя на языке UI.
- Регрессионный тест: плейсхолдер project_id → submit НЕ вызывается, ответ
  содержит «не настроен».
