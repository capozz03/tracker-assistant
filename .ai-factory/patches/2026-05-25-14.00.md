# Теги не отображались: Claude возвращает имена вместо UUID + некорректный формат описания

**Date:** 2026-05-25 14:00
**Files:** scripts/submit_task.py, tests/test_submit_task.py
**Severity:** medium

## Problem

1. Тег добавлялся в задачу (PATCH 200 OK), но в UI Timetta отображался как серый кружок без текста.
2. Описание задачи использовало заголовки `##` вместо `#### ***...***`.
3. Названия задач содержали префиксы «Frontend:»/«Backend:», хотя теги уже указывают слой.

## Root Cause

**Тег без отображения:** `_format_tags` принимает строки и создаёт `{directoryEntryId: value}`.
Если Claude возвращает имя тега (`"Фронтенд"`, `"backend"`) вместо UUID — создаётся невалидный
`directoryEntryId`. Timetta принимает PATCH (не валидирует формат UUID), но не может разрешить
имя чипа в UI. Отображается пустой чип с кнопкой удаления.

Причина: промпт давал список тегов как `id=<uuid> name=<name>` и просил использовать UUID,
но Claude — особенно при русскоязычных тегах — периодически возвращал имена.

**Формат заголовков:** В `_PROMPT_TEMPLATE` не было инструкции о форматировании заголовков.
Claude по умолчанию использовал `##`.

**Слой в названии:** Промпт явно предписывал `"[Слой]: <суть>"` — противоречило требованию
не дублировать слой, уже отражённый в тегах.

## Solution

1. Добавлена функция `_resolve_tags(tag_values, known_tags)`:
   - Строит три справочника: UUID passthrough, `name.lower() → id`, `code.lower() → id`
   - Принимает любой формат (UUID, имя, код — любой регистр), возвращает только валидные UUID
   - Логирует преобразования с префиксом `[FIX-TAG]`
   - Пропускает нераспознанные значения с `WARNING`

2. В `submit_requirements()` вызов `_resolve_tags(raw_tags, tags)` ПЕРЕД передачей в адаптер.

3. Обновлён `_PROMPT_TEMPLATE`:
   - Убраны «Frontend:»/«Backend:» префиксы в `summary`
   - Добавлена инструкция по заголовкам: `#### ***Название раздела***`

## Prevention

- При передаче данных из LLM в API — всегда делать resolving к каноническим идентификаторам.
  Не доверять LLM в корректности UUID: добавлять lookup-слой.
- В промптах для Claude: явно указывать FORMAT полей (заголовки, структуру описания).
- Тест: добавить `TestResolveTagsFunction` (11 кейсов) — покрывает UUID passthrough,
  name/code lookup (case-insensitive), смешанный ввод, пустые входы.
