# Telegram-бот: интерактивные кнопки, выбор проекта из API, избранные

**Date:** 2026-05-25 18:45
**Files:** src/tracker_assistant/telegram/handlers.py, tests/test_telegram_handlers.py, docs/telegram-bot.md
**Severity:** medium

## Problem

Бот не имел интерактивных элементов: `/start` возвращал статичный текст, `/project` показывал
данные из env/конфига вместо реальных проектов Timetta, `/tasks` показывала session-only ID задач
без практической ценности.

## Root Cause

Обработчики были написаны как MVP без inline-клавиатур. `cmd_project` читал только `ProjectRegistry`
(загружается из `telegram_projects.json` / `.env`) — прямого запроса к Timetta API не было.
`cmd_tasks` хранила `last_task_ids` из `context.user_data`, что теряется при рестарте и не несёт
ценности вне сессии прикрепления файлов.

Отсутствовал `CallbackQueryHandler` — нажатия на inline-кнопки не обрабатывались.

## Solution

1. **`_active_project` helper** — возвращает `ProjectConfig` из `context.chat_data["active_project_id"]`
   (динамический выбор) или fallback к `registry.get_project(chat_id)`. Используется в `handle_text`
   и `_handle_media_message` вместо прямого вызова registry.

2. **`_fetch_project_content` inner helper** — запрашивает `adapter.get_projects()` через
   `asyncio.to_thread`, строит inline-клавиатуру, кеширует `{id: name}` в `context.chat_data["_project_cache"]`.
   Переиспользуется в `cmd_project` и `handle_callback(action:projects)`.

3. **`cmd_start`** — добавлены 2 inline-кнопки: «📋 Выбрать проект» и «⭐ Избранные».

4. **`cmd_project`** — вызывает `_fetch_project_content`, показывает список как inline-клавиатуру;
   активный проект помечается `✅`.

5. **`cmd_favorites`** (замена `cmd_tasks`) — показывает `user_data["favorites"]` с кнопками
   `⭐ Выбрать` и `✖ Убрать`.

6. **`handle_callback`** — единый `CallbackQueryHandler` для паттернов:
   - `sel:{uuid}` — сохраняет `active_project_id` в `chat_data`, предлагает добавить в избранные
   - `fav_add:{uuid}` — добавляет в `user_data["favorites"]` (без дублей)
   - `fav_rm:{uuid}` — удаляет из `user_data["favorites"]`
   - `action:projects` / `action:favorites` — переходы между видами через edit_message_text

## Prevention

- При добавлении inline-кнопок — сразу добавлять `CallbackQueryHandler` в `make_handlers`.
- `callback_data` ограничен 64 байтами: предпочитать `{prefix}:{uuid}` (40-44 байта) и хранить
  отображаемые данные в `chat_data["_project_cache"]`, а не в `callback_data`.
- `context.chat_data` и `context.user_data` — in-memory. Для персистентности (избранные между
  сессиями) потребуется `persistence` в `ApplicationBuilder`.
- Тесты: при добавлении нового хендлера перед существующими — обновлять индексы в тестах.
  Лучше использовать `handlers[-1]` для последнего и семантические комментарии для промежуточных.
- `make_mock_context()` должен включать `chat_data = {}` (dict), иначе `isinstance(context.chat_data, dict)`
  возвращает False и динамический выбор проекта не работает в тестах.
